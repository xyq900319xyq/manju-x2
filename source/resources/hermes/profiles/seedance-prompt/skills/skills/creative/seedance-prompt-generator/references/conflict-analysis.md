# 两份知识文档逐字段冲突对照

本文档逐行对比 Director Angel 编译格式 与 工业化制片架构师设定，标出所有差异点供决策。

---

## A. 整体输出结构

| # | Director Angel（现有 SKILL.md + director-angel-format.md） | 工业化制片（seedance-industrial-architect.md） | 差异性质 |
|---|---|---|---|
| A1 | 一个 Group 一段完整输出，内部 3-5 镜，时长可变 | 一个 Segment 一段独立输出，固定 5 镜 × 3秒 = 15秒 | **冲突**：镜头数/时长是否固定 |
| A2 | 输出纯文本，无 emoji 标题 | 每段开头带 🎞️ emoji + Segment 标题 | 差异 |
| A3 | 先全局（风格硬规则/导演校准/改编事实层），再逐镜 | 先全局（Asset Definitions/Global Style），再 Matrix，再 Audio | 结构顺序不同 |
| A4 | 有 Execution profile 行：`--resolution 720p --ratio 16:9 --duration 12 --camera_fixed false` | 无 Execution profile 行 | **冲突**：是否需要 CLI 参数行 |
| A5 | 输出前无「导演核算报告」 | 输出前先发 🎬 导演核算报告（总时长预估 + 拆段方案） | **冲突**：是否需要前置核算报告 |

## B. 对白处理（最核心冲突）

| # | Director Angel | 工业化制片 | 差异性质 |
|---|---|---|---|
| B1 | **对白动作化**：台词不允许出现在提示词中，必须转化为可视化表演 | **对白保留原文**：台词全文写入，打 `[Lip-sync]` 标签 | **根本冲突** |
| B2 | 示例：说"我不会原谅你"→「缓慢摇头，后退半步，眼神从愤怒转为空洞」 | 示例：`[主体动作][Lip-sync] 何丹冷笑："违约金从期权扣。"` | |
| B3 | 无音频模块 | 有独立 `[Native Audio]` 区块，列出所有 SFX + 对白全文 | **冲突**：是否需要音频模块 |
| B4 | 旁白无特殊处理 | 旁白标注 `Voice-over` / `O.S`（画外音） | 差异 |
| B5 | 无「反应镜头+画外音承接长对白」规则 | 长对白必须配合 Reaction Shots + Voice-over 跨段流动 | 差异 |

## C. 顶层模块（Segment/Group 级别的包装）

| # | Director Angel | 工业化制片 | 差异性质 |
|---|---|---|---|
| C1 | `提交级视觉风格硬规则`：风格描述 + Board 参考图优先级声明 | `[Global Style]`：纯粹的视觉风格描述，无 Board 图优先级逻辑 | **冲突**：是否有 Board 参考图约束 |
| C2 | 风格值为项目级变量：「新海诚风格、电影感、高质量、4K」 | 风格为全局硬锁定：「低饱和、高对比、写实」 | **冲突**：风格内容是否可变 |
| C3 | `Director Angel guidance:`（空占位符） | 无此字段 | 差异 |
| C4 | `Base compiled prompt:` 一段完整的逐镜文字描述 | 无对应字段（直接进入 Matrix） | 差异 |
| C5 | `导演校准要点`：组间承接策略、动作因果、同一空间约束 | 无对应字段 | **冲突**：是否需要导演校准 |
| C6 | `组间承接建议：当前组默认独立提交` | 无对应字段 | 差异 |
| C7 | `【单镜出镜控制】- 出镜表` | 无独立出镜表（出镜信息分散在每个 Shot 的「主体动作」中） | **冲突**：出镜控制方式 |
| C8 | `【标准分组连续构图锁定】`：构图基准/硬锁位/轴线规则/站位摘要 | 无对应字段（靠视觉桥梁 + 「延续前段」标注实现） | **冲突**：连贯性机制的精度 |
| C9 | `片段级改编事实层`：空间建立→动作爆发→连续性控制→结果落点 | 无对应字段 | **冲突**：是否需要改编事实层 |
| C10 | `[Asset Definitions]` 无此字段 | `[Asset Definitions]`：列出 @image1, @image2... 标签 | **冲突**：是否需要资产映射模块 |
| C11 | 场景用中文名：「废弃工厂」 | 场景用 @image 标签引用 | 差异 |
| C12 | 无「核心剧情摘要」 | `核心剧情摘要` 作为独立模块 | 差异 |

## D. 单镜/单 Shot 字段逐项对照

| 字段 | Director Angel | 工业化制片 | 冲突？ |
|---|---|---|---|
| D1 镜头标识 | `镜头N [Xs-Ys]「场景名称」` | `Shot N (Xs-Ys)` | 差异：是否有场景名 |
| D2 出镜角色 | 独立字段，含主次标注：「林星野（主）-红发金瞳」 | 无独立字段，嵌入「主体动作」中：「何丹抱着纸箱」 | **冲突**：出镜角色的形式 |
| D3 景别 | 标准缩写+中文：`ELS/LS/MS/MCU/CU/ECU` | 自由中文：「广角推近」「中景侧拍」「极致特写」「瞳孔微距」 | **冲突**：景别命名体系 |
| D4 运镜 | 分类体系：固定机位/横摇/竖摇/推拉/跟拍/升降/手持 | 分类不同：Handheld breathing / Snap Zoom / Orbit / Reflexive Jitter / Dolly/Pan/Truck / Focus Pull | **冲突**：运镜分类体系不同 |
| D5 特殊技法 | 独立字段：J-Cut/L-Cut/浅景深/慢镜头/Dutch Angle/POV | 嵌入镜头语言字段：「视觉桥梁：机位定格在侧方仰拍」 | 差异 |
| D6 镜头职责 | 枚举字段：动作起势/氛围铺垫/情绪爆发/信息交代/转折过渡/余韵收束 | 无此字段 | **冲突**：是否需要镜头职责 |
| D7 表演节点 | 独立字段：具体动作/表情/肢体语言 | 无独立字段，融合在「主体动作」中 | 差异 |
| D8 动作推进 | 独立字段：从状态A→状态B的转变描述 | 无独立字段 | **冲突**：是否需要动作推进 |
| D9 叙事职责 | 枚举字段：action/动作起势/氛围铺垫/情绪爆发/信息交代/反应镜头 | 用 `[Reaction]` 标签：「[主体动作][Reaction] 镜头切向霓虹大楼」 | **冲突** |
| D10 组织要求 | 节奏字段：`build/release/hold/sustain` | 无此字段 | **冲突**：是否需要节奏控制 |
| D11 物理细节 | 无此字段 | 独立字段，量化数值：「0.2s 弹起」「20cm 水花」「自由落体引力模拟」 | **冲突**：是否需要量化物理 |
| D12 对白标注 | 无（对白已被动作化） | `[Lip-sync]` 标签标注说话动作 | **冲突**（与 B1 同源） |

## E. 连贯性/跨段衔接

| # | Director Angel | 工业化制片 | 差异性质 |
|---|---|---|---|
| E1 | `【标准分组连续构图锁定】` 含 4 个子项 | 视觉桥梁：Segment[N].Shot1 锁定 Segment[N-1].Shot5 | 精度不同 |
| E2 | 构图基准：镜头2继承镜头1的摄像头位置、FOV | 无 | **冲突** |
| E3 | 硬锁位：目标角色在画面九宫格位置 | 无 | **冲突** |
| E4 | 轴线规则：对话轴线180°，动作轴线方向 | 无 | **冲突** |
| E5 | 站位摘要：角色相对位置（距机器2m） | 无 | **冲突** |
| E6 | 同组内镜头间连续性描述 | 「视觉保戏」：Segment[N].Shot1 复述 Segment[N-1].Shot5 结尾状态 | 两种机制 |
| E7 | 无「延续前段」标注 | 「延续前段」标注在 Shot 1 的 主体动作前 | 差异 |

## F. 时长 / 分片逻辑

| # | Director Angel | 工业化制片 | 差异性质 |
|---|---|---|---|
| F1 | 每镜时长可变（如 0s-3s, 3s-7s, 7s-12s） | 每镜固定 3秒 | **冲突**：单镜时长 |
| F2 | 每组 3-5 镜，总时长 5-12 秒（建议值） | 每段固定 5 镜，总时长 15 秒（硬性） | **冲突**：每段镜数和总时长 |
| F3 | 拆组触发：硬切、场景转换、时间跳跃、超 5 镜或超 15 秒 | 拆段触发：纯时长（≤15=1段, 16-30=2段） | **冲突**：拆组依据 |
| F4 | 时长来源：分镜表中各镜头时长直接相加 | 时长来源：对白字数÷3.5 + 动作数×2-3秒（从剧本估算） | **冲突**：时长计算方式 |
| F5 | 无超长场景提示 | 超 500 字/10 段时触发：「检测到场景极长，是否分步生成？」 | **冲突**：是否需要交互确认 |

## 汇总：需要你做决策的关键问题

**Q1（最核心）** 对白处理：保留原文+[Lip-sync]+Native Audio 区块，还是转化为可视化表演（对白动作化）？

**Q2** 输出模块结构：用哪套模块组合？要不要 Execution profile 行？

**Q3** 单镜字段：保留 Director Angel 的 7 字段，还是工业化制片的 3 字段，还是融合？

**Q4** 镜头数/时长：固定 5镜×3秒=15秒，还是可变？

**Q5** 分片依据：按叙事（硬切/场景转换），还是按纯时长？

**Q6** 风格：锁定「低饱和高对比写实」，还是从资产设计灵活提取？

**Q7** 连贯性机制：Director Angel 的 4 项空间级锁定，还是工业化的「视觉桥梁」？

**Q8** 物理量化：要不要「0.2s 弹起」「20cm 水花」这种数值？

**Q9** 景别/运镜术语体系：用哪套？

**Q10** 资产引用：用 @image 标签还是文字描述角色外貌？

**Q11** 语言策略：全中文，还是中英混合？
