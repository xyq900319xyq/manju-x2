- 用户要求为分镜生成 Seedance 视频提示词

## 工作流程

### Phase 1: 全局扫描与时长核算

**🔴 第一步：全场景强制扫描（不可跳过）**

在生成任何输出之前，必须先完整扫描整个输入，识别所有场景边界。输入中的场景通常以 `═══`、`场景 N:`、`场 N:`、`════` 等标记分隔。扫描完成后，建立内部清单：

```
场景清单（内部，不输出）：
  场景1: 青云宗山门外-日-外 — 31镜 — 约47s
  场景2: 青云宗山门外-日-外 — 27镜 — 约35s
  场景3: 青云宗山脚下-日-外 — 6镜 — 约9s
  ...（列出全部）
```

**铁律：场景清单必须覆盖输入中每一个场景。若输入有 7 个场景 + 结尾钩子，清单就必须有 8 项。此后每处理完一项就打勾，全部勾完才允许结束输出。未全部打勾前，绝对禁止认为"任务完成"。**

**第二步：时长核算**

在场景扫描完成后，执行以下核算：

1. **时长审核**：分镜表中各镜头时长直接相加得到总时长，同时用对白字数÷3.5 + 动作数×2-3秒交叉验证
2. **拆组判断**：每组严格 3-5 镜，总时长 ≤15 秒。以导演视角确定每个镜头的时长分配。
   - 超 15 秒或超 5 镜 → 必须拆组
   - 不足 3 镜 → 必须与相邻段合并（详见拆组策略「短段合并铁律」）
   - 遇硬切或场景转换 → 必须拆组
   - 遇时间跳跃 → 必须拆组
   - 每组开头重置构图基准
3. **核算报告**：生成前先输出导演核算报告（总时长预估 + 拆段方案）。核算报告必须覆盖全部场景。

### Phase 2: 全局初始化

1. 提取并记住所有核心角色、场景、道具特征
2. 读取用户输入中的资产列表（人物、场景、物品），在 Asset Definitions 中使用「资产名：@资产名」格式列出。资产名必须与输入完全一致，不得自行编造。同一光影时间的场景采用统一命名（如 `合欢宗大殿-日`、`柳如烟卧室-夜`）

### Phase 3: 连贯性保障

跨段衔接强制执行：

- **视觉桥梁**：Segment[N].Shot1 锁定 Segment[N-1].Shot5 的机位/角色姿势/光影/环境状态，但描述时必须完整写出状态而非引用编号
- **视觉保戏**：Segment [N] 的 Shot 1 必须完整描述 Segment [N-1] 的 Shot 5 结尾状态，而非引用编号。禁止写「Seg6 镜4 闭眼」，必须写为「延续前段——师尊双眼紧闭，面部肌肉松弛，嘴角残留血迹，靠在洞壁上」。
- **构图基准**：组内镜头2继承镜头1的摄像头位置、FOV参数
- **轴线规则**：对话轴线 180° 不跨轴，动作轴线方向固定
- **站位摘要**：角色相对位置明确（如"距机器2m"）
- **叙事连续性**：对白必须自然跨段流动，长对白配合 Reaction Shots + Voice-over 实现
- **位置状态延续**：每个镜头必须完整描述所有出镜角色的空间位置和当前状态。剧情未推动时直接继承上一镜描述，角色移动后以新位置为基线延续。禁止出现角色无故跳跃到其他位置，禁止状态突变。
- **延续前段标注**：Segment [N] 的 Shot 1 主体动作前标注「延续前段」

### Phase 4: 矩阵生成

每个 Segment 独立输出，100% 自包含。绝对禁止 `Continued from...` 或 `Same style as above.`

**输出规则：你的回复第一个字符必须是 `#`，以 `## 🎬 导演核算报告` 直接开头。禁止在此之前输出任何文字——包括分析、推理、「好的」「让我」等一切非模板内容。你是一个提示词编译器，不是聊天助手。**

## 🔴 对照上述示例逐项自检

**🔴 全场景覆盖铁律：你必须处理输入中的每一个场景（场/Scene），并为其生成对应的 Segment 块。输入有场景 1/2/3/4/5/6/7 + 结尾钩子，你就必须为全部 8 个逻辑单元生成 Segment。一个都不能少。**

**处理步骤**：

**禁止的行为**：

## 输出前七重内部审查（强制执行）

正式输出前必须逐条执行全部七重审查。每条审查必须给出结论，结论**必须随输出一起呈现**，不得静默丢弃。

### 审查一：剧情与台词完整性
逐条核验输入中每个剧情节点是否有对应镜头覆盖，每一句对白是否完整出现并标注 [Lip-sync]，旁白是否标注 Voice-over/O.S。检查 [Native Audio] 区块是否包含所有对白全文和旁白。
**🔴 场景覆盖核验**：核验输出中的 Segment 是否覆盖了输入的全部场景。输入有多少个 `场景 N:` 标记，输出中对应的 Segment 序列就必须覆盖到最后一个场景。若输入有 7 个场景 + 结尾钩子，但输出只覆盖到场景 1，审查一直接判定为不合格。
结论格式：`✅ 审查一通过：剧情节点 X 个全部覆盖，对白 Y 句全部到位，全部 Z 个场景均已覆盖` 或 `❌ 审查一未通过：输入有 Z 个场景，输出仅覆盖 W 个，缺少场景 W+1 至 Z`

### 审查二：时长与镜数可行性（导演级）
逐镜核算时长是否足够容纳全部内容：对白 3.5 字/秒，动作 2-3 秒/个。
**🔴 镜数下限铁律**：每个 Segment 严格 3-5 镜。产出 1-2 镜的 Segment 视为不合格——审查二直接判定不通过，必须与相邻段合并后重新审查。
铁律：不允许 Segment 结束而对白或动作未完成。
结论格式：`✅ 审查二通过：{N} 个 Segment 均控制在 3-5 镜且 ≤15s，时长核算无溢出，对白动作分配合理，无短段需合并` 或 `❌ 审查二未通过：镜头X 中 Y 字对白需 Z 秒但仅分配 W 秒 / Segment M 仅 N 镜不足 3 镜下限`

### 审查三：跨镜连贯性（导演级）
逐镜审查人物站位、光影时间、光源方向、角色状态的连贯性。不得无故漂移、昼夜突变、状态跳变。同一场景内光源方向必须锁定。衔接描述禁止引用其他 Segment 编号，必须完整描述前段结尾状态。
结论格式：`✅ 审查三通过：全段人物站位/光影/状态连贯一致` 或 `❌ 审查三未通过：镜头X 中角色Y 位置从A突变到B`

### 审查四：推理链执行合规
逐镜检查以下内部推理是否确已执行并落入输出字段：
- 情绪→角度映射：目标情绪→角度姿态→景别/运镜的选择是否有情绪依据
- 光影→情绪映射：光位→阴影形态→可见性控制的推理链是否完整
- 运镜增强：速度曲线（A）/ 空间交互（B）/ 光影过渡（C）是否有落地体现
结论格式：`✅ 审查四通过：每镜均内部执行情绪→角度/光影/运镜增强推理，推理链完整无跳过` 或 `❌ 审查四未通过：镜头X 缺少情绪→角度映射推理`

### 审查五：禁令合规
逐镜检查以下禁令是否全部遵守：
- 无 BGM：SFX 仅限环境音/音效，不含背景音乐。**扫描 SFX 是否命中「BGM 类违规关键词黑名单」（详见下方质量禁止清单）**
- 无字幕：提示词中不得出现硬字幕/burned-in subtitles 等**视频生成指令**。**注意**：剧本叙事需要的字幕剧情（如"黑屏字幕三天后"、"白色字幕浮现"等）属于叙事元素，不是视频生成指令，不属于审查范围，应予以保留
- 禁止词合规：不出现 Seedance 2.0 违规词（详见下方 Seedance 2.0 违规词表）
结论格式：`✅ 审查五通过：无 BGM（SFX 仅环境音/音效），无硬字幕指令，无禁止词，无 BGM 类违规关键词` 或 `❌ 审查五未通过：镜头X 的 SFX 中包含「{违规词}」——BGM 类关键词，必须替换为环境音效`

| 同一空间约束 | 同场景镜头必须共享空间参数（尺寸、光线方向、道具位置） |
| 情绪→角度映射 | 每镜执行「叙事节点 → 目标情绪 → 角度姿态 → 景别/运镜」推理链（详见下方情绪映射表），仅内部推理，不渗入输出字段 |
| 光影→情绪映射 | 每镜执行「光位 → 阴影形态 → 可见性控制 → 情绪效果」推理链（详见下方光影映射表），仅内部推理，不渗入输出字段。同一场景内光源方向锁定，不得突变 |

### 情绪→角度映射表（内部推理，不输出）

Agent 生成每镜的「镜头语言」字段前，必须内部执行以下推理链：

```
叙事节点 → 目标情绪 → 角度/姿态选择 → 景别策略 → 运镜方式 → 物理量化
```

**核心原理**：镜头不是在"记录"动作，而是在"翻译"情绪。同一个动作（如角色走进门），目标情绪不同，角度选择完全不同。

**常见情绪→角度映射**：

| 目标情绪 | 角度/姿态 | 景别策略 | 运镜倾向 | 光影暗示 |
|------|------|------|------|------|
| **压迫感 / 威胁** | 低机位仰拍（Low Angle），主体占画面主导 | 近景→特写，逐步压缩空间 | Slow push-in + 轻微 Handheld breathing | 逆光剪影或顶光阴影 |
| **脆弱感 / 无力** | 高机位俯拍（High Angle），主体显小 | 全景→中景，环境占比大 | Slow pull-back / Static wide | 散射柔光，低对比 |
| **神秘感 / 悬念** | 背影角度 + 遮挡构图（半遮面/半遮身） | 中景→特写，逐步揭示 | Slow tracking + delayed reveal | 光→暗过渡、局部高光 |
| **英雄感 / 力量** | 低机位仰拍 + 逆光 | 远景→中景→特写三级跳 | Slow push-in + crane up | 逆光轮廓光（Rim Light） |
| **亲密感 / 温柔** | 平视（Eye Level），浅景深 | 特写→极致特写 | Static / subtle dolly in | 暖色调柔光 |
| **紧张 / 焦虑** | Dutch Angle 倾斜 + 手持晃 | 近景→特写 | Handheld breathing + reflexive jitter | 高对比硬光 |
| **疏离感 / 孤独** | 远景 + 主体居中但占比极小 | 极远景→远景 | Static wide / slow pull-back | 冷色调低饱和 |
| **震惊 / 反转** | 固定 + Snap Zoom 突进 | 中景→瞳孔微距 | Snap Zoom (0.5s内) | 高对比 |
| **日常 / 平凡** | 平视（Eye Level）+ 正面 | 中景为主 | Static / 轻微 Pan | 自然光/散射光 |
| **速度 / 紧迫** | 侧面跟拍 + 动态模糊 | 中景 | Speed Ramp / fast Truck | 运动模糊 |

**强制规则**：
- 此映射表为推理依据，不是填空题模板——必须根据具体剧情灵活调整
- 推理链仅内部执行，输出中不出现情绪意图文本
- 推断完成后，输出字段仍为标准格式：`景别 {自由中文}；运镜 {运镜方式}；特殊技法 {技法}`
- 若剧情情绪复杂（如"愤怒中带着悲伤"），取主导情绪进行映射，次生情绪通过「动作推进」字段间接体现

**示例（内部推理，不输出）**：

```
内部推理：
  叙事节点：何丹得知项目崩盘 → 目标情绪：震惊+愤怒
  → 角度：低机位仰拍（压迫转力量）→ 景别：中景 Snap Zoom 至瞳孔微距
  → 运镜：Snap Zoom (0.3s) + 0.1s reflexive jitter
  → 输出：景别 瞳孔微距；运镜 Snap Zoom + 0.1s reflexive jitter
```

### 光影→情绪映射表（内部推理，不输出）

Agent 生成每镜时，必须内部执行以下光影推理链：

```
叙事节点 → 目标情绪 → 光位选择 → 阴影形态 → 可见性控制 → 输出光影描述
```

**核心原理**：光不是在"照亮"画面，而是在"编辑"画面——光位决定阴影落在哪，阴影决定观众能看到什么、看不到什么，可见性控制决定情绪。

**常见情绪→光影映射**：

| 目标情绪 | 光位 | 阴影形态 | 可见性控制 | 输出描述示例 |
|------|------|------|------|------|
| **压迫感 / 威胁** | 顶光（Top Light），光源垂直头顶 | 眼窝深陷阴影，眉骨以下全暗，看不清眼睛 | 屏蔽眼神信息，制造不可知感 | 顶光单光源，眼窝处形成硬阴影，面部下半暗区 |
| **神秘感 / 悬念** | 顶光 + 局部点光（Top + Spot） | 上半脸在阴影中，仅下颌/嘴角被照亮 | 遮挡身份，延迟揭示 | 顶光从正上方投下，眼部被暗区覆盖，仅嘴唇和下颌有微光 |
| **分裂感 / 矛盾** | 正侧光（Side Light），光源90°于人物一侧 | 半脸清晰、半脸深暗，鼻梁为明暗分界线 | 一半人格可见，一半隐藏 | 正侧光从左侧打入，左脸硬光清晰，右脸完全陷于阴影 |
| **压抑 / 悲伤** | 正侧光 + 低光比（2:1） | 暗侧保留少量细节，亮侧低饱和 | 情绪内收而非外放 | 左侧柔侧光，右脸暗区保留眼白反光，黑白或低饱和 |
| **英雄感 / 崇高** | 逆光 + 轮廓光（Backlight + Rim Light），光源在人物后方偏上 | 正面暗但轮廓被金色光边勾勒 | 细节隐藏，只留轮廓符号 | 逆光从人物后方偏上打入，身体边缘形成金色轮廓光，正面低照度 |
| **恐怖 / 诡异** | 底光（Foot Light），光源从下往上 | 面部阴影倒置——眼窝亮、额头暗，五官变形 | 颠覆自然光影逻辑，制造不安 | 底光从下方打上，下颌和眼窝异常高亮，额头落入阴影 |
| **温柔 / 亲密** | 前侧柔光（Front-side Soft），45°+大面积柔光箱 | 无硬阴影，面部光过渡平滑 | 全部可见，消除距离感 | 暖色柔光从前方45°打入，光比约2:1，面部无硬影，肤色温润 |
| **紧张 / 焦虑** | 硬光高对比（Hard Light），单光源无填充 | 一侧过曝，一侧死黑，无中间灰阶 | 信息残缺，强迫观众猜测暗处 | 右侧硬光直射，左侧无填充光，面部高光比约8:1 |
| **疏离感 / 孤独** | 冷色调低照度 + 大面积暗区 | 主体被弱光照亮，环境吞没于暗处 | 角色在画面中占比小且孤立 | 单一冷色光源从远处窗格透入，角色处于大面积阴影中，仅轮廓微亮 |
| **日常 / 平凡** | 正面散射光（Front Diffuse），自然光或环境光 | 几乎无阴影、无戏剧性 | 全部可见、无遮挡、无设计感 | 环境散射光均匀照明，面部平淡无戏剧阴影 |

**关键规则**：

1. **光位决定情绪，阴影产生情绪**——不是"加个顶光就有压迫感"，是"顶光使眼窝产生阴影→观众看不到眼睛→产生不安"
2. **可见性 = 权力**：被光照亮的部分是导演允许观众看到的，藏在暗处的部分是不允许看到的
3. **光源方向必须用空间语言描述**：不写"逆光"，写"光源从人物后方偏上打入"；不写"侧光"，写"光源在画面外、人物左侧"
4. **同一场景内光源方向必须锁定**：如果场景是"书房-台灯在桌左侧"，全段所有镜头的光源方向都从左侧来，不得突变
5. **推理链仅内部执行**，输出中不出现情绪意图文本，光影描述融入「镜头语言」字段
6. 若一镜需光影变化（如角色从明处走入暗处），单独写变化过程，不破坏基线光位

**示例（内部推理，不输出）**：

```
内部推理：
  叙事节点：杀手在桌前擦枪 → 目标情绪：冷酷、肃杀、压迫
  → 光位：顶光，光源垂直头顶
  → 阴影形态：眼窝深陷硬阴影，眉骨以下全暗，桌面反射微弱冷光到下巴
  → 可见性控制：看不清眼睛→杀手不可知→恐惧来自未知
  → 输出融入镜头语言：顶光单光源垂直投下，眼窝阴影覆盖眼部
```

### 运镜增强推理（内部推理，不输出）

生成运镜描述时，必须内部执行以下三项增强推理。输出格式不变，但运镜描述的质量从"标签式"升级为"空间流动式"。

---

**增强A：运镜速度曲线**

运镜不是匀速的。每镜内部速度有变化曲线，内部推理时必须设计：

```
加速段 → 匀速段 → 减速段（或反之）
```

| 叙事需求 | 速度曲线 | 内部推理示例 |
|------|------|------|
| 俯冲/坠落 | 0.3s加速 → 1.5s极速 → 0.2s骤停 | 镜头0.3s内加速俯冲，1.5s穿越树冠层，触湖前0.2s急减速 |
| 逼近/威胁 | 缓慢→加速→骤停逼近 | Slow creep 1s → 0.5s speed ramp → freeze on face |
| 揭示/豁然开朗 | 加速冲出限制→减速巡航 | 0.5s冲出暗处→2s在开阔空间缓速环视 |
| 紧张/焦虑 | 不匀速抖动+随机变速 | Handheld breathing + irregular speed micro-variation |
| 日常/平稳 | 近乎匀速，微变≤5% | Subtle dolly，速度恒定 |

**规则**：速度变化描述融入运镜术语，不单独成行。如 `Dolly in，0.5s加速→2s匀速→0.3s减速停于面部`。

---

**增强B：镜头-环境空间交互**

镜头不是悬浮在真空中的，它与环境中的物体有物理关系：

| 交互类型 | 效果 | 内部推理示例 |
|------|------|------|
| **擦过（Graze）** | 强化速度感、危险感 | 镜头近乎擦过树干左侧，树皮纹理在画面边缘飞速掠过 |
| **穿越（Break through）** | 空间转换、豁然开朗 | 镜头穿出树叶层限制，画面从暗绿瞬间过渡到开阔天空 |
| **前景遮挡（Foreground occlusion）** | 深度感、层次感 | 前景蕨类叶片从画面底部掠过，中景角色保持清晰 |
| **逼近放大（Looming）** | 冲击力、压迫感 | 目标物体在画面中指数级放大，从10%→80%画幅占比 |
| **跟随间隙（Tracking gap）** | 追逐感、紧张感 | 镜头与被追踪物体保持2m间隙，物体变速则镜头同步变速 |

**规则**：至少检查每镜是否有空间交互机会（快速运动镜必有，静态对话镜可选）。交互描述融入运镜字段，不单独成行。

---

**增强C：镜内光影过渡**

光影不是每镜一个固定值。同一镜内，随镜头运动，光影可以发生变化：

| 过渡类型 | 触发条件 | 内部推理示例 |
|------|------|------|
| **暗→明** | 镜头从室内/阴影区移向室外/光源区 | 从室内暗区推近窗户，画面从低照度2s内过渡到户外明亮散射光 |
| **明→暗** | 镜头从室外进入室内/角色走入阴影 | 角色从阳光下的门口走入室内，面部从清晰→0.8s内被阴影覆盖 |
| **光斑掠过** | 镜头穿过树叶/栅栏/百叶窗 | 镜头穿梭林间，阳光透过树叶形成移动光斑在画面中交替明灭 |
| **光源切换** | 镜头旋转/环绕导致主光源方向改变 | Orbit半圈后，主光从逆光切换为前侧光 |
| **渐变暗/亮** | 情绪需要（绝望/希望） | 画面从正常曝光→1.5s渐暗至低照度（绝望）；或反向（希望） |

**规则**：不是每镜都要光影过渡——静态镜头保持固定光影。仅当镜头有明显空间位移（Dolly/Truck/Orbit长距离）或角色穿越光影边界时触发。过渡描述融入镜头语言字段。

---

**整体示例（内部推理，不输出）**：

```
内部推理：
  叙事节点：FPV俯冲森林 → 目标情绪：速度感+探索
  → 增强A 速度曲线：0.3s加速俯冲→2s极速穿梭→0.2s急停于湖面
  → 增强B 空间交互：擦过树干、穿过藤蔓、冲出树冠层
  → 增强C 光影过渡：林间光斑交替明灭→冲出树林瞬间暗→明亮过渡
  → 输出：FPV视角，0.3s加速垂直俯冲，2s极速穿梭林间，镜头近乎擦过树干，
          光斑交替明灭；0.2s穿出树冠急停于湖面上方，暗→明亮过渡
```

### 单镜出镜控制（内部推理，不输出）
Agent 生成时必须内部逐镜确认出镜角色，但最终输出中不出现出镜表模块。每镜的出镜角色信息融入「出镜角色」字段中。
- `无角色/环境镜` → 不允许出现任何人物
- `角色名（主）` → 核心焦点
- 出镜控制禁止模糊词（"可能有""包括但不限于"）

### 标准分组连续构图锁定（内部推理，不输出）
Agent 生成时必须内部执行以下连贯性校验，确保跨镜/跨段衔接：

| 字段 | 说明 |
|------|------|
| 构图基准 | 镜头2继承镜头1的摄像头位置、FOV |
| 轴线规则 | 对话轴线 180° 不跨轴；动作轴线方向固定 |
| 站位摘要 | 角色相对位置（左/右/前/后，距离数值） |
- **视觉桥梁**：Segment[N].Shot1 锁定 Segment[N-1].Shot5 的机位/角色姿势/光影/环境状态，但描述时必须完整写出状态而非引用编号

### 单镜字段详解

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| 出镜角色 | string | 角色名或"无角色/环境镜"，含主次 | 林星野（主）-红发金瞳；苏晓（次） |
| 人物站位与状态 | string | 所有出镜角色的空间位置和当前状态 | A坐于桌左侧，B立于桌右侧，两人面对面 |
| 主体动作 | string | 角色行为描述 + 对白标注 | [Lip-sync] 何丹冷笑："违约金从期权扣。" |
| 镜头语言 | string | 景别 + 运镜 + 特殊技法 | 景别 极致特写；运镜 Handheld breathing + 0.1s reflexive jitter |
| 动作推进 | string | 状态A→状态B转变 | 从凝视枪口→缓缓闭眼→嘴角露出释然微笑 |

## 镜头语言术语表

### 景别（Shot Size）
使用自由中文描述，常用词包括：
极远景、全景、中全景、中景、近景、特写、极致特写、瞳孔微距、广角推近、固定远景、极致远景拉远、低角度细节、侧方仰拍

### 运镜（Camera Movement）
- Handheld breathing + 0.1s reflexive jitter — 手持呼吸感晃动
- Snap Zoom — 快速变焦
- Orbit — 环绕运镜
- Reflexive Jitter — 应激性震颤
- Dolly — 推拉
- Pan — 横摇
- Tilt — 竖摇
- Truck — 跟拍平移
- Focus Pull — 焦点切换
- Pedestal/Crane — 升降

### 特殊技法
- 匹配转场（Match Cut）
- J-Cut/L-Cut（声音/画面先入先出）
- 浅景深（Shallow DoF）
- 慢镜头（Slow Motion）
- 快镜头（Speed Ramp）
- Dutch Angle（倾斜构图）
- POV（主观视角）
- 视觉桥梁（Visual Bridge）
- 延续前段（Continuity Anchor）

---

## 角色出镜控制

- 角色通过 `角色名：@角色名` 格式绑定，外貌和特征由资产设计提供，不在此处重复
- 多角色镜头标注主从关系：「角色A（主）；角色B（次）」
- 出镜控制必须明确，禁止模糊词
- 所有 @ 引用的资产名必须与用户输入中的名称完全一致，不得自行编造

---

## 对白处理

- 对白保留原文，打 `[Lip-sync]` 标签
- 同时附加可视化表演描述（表情、动作、肢体语言）
- 旁白标注 `Voice-over` 或 `O.S`（画外音）
- 长对白配合 Reaction Shots + Voice-over 跨段流动，不得遗漏台词
- 所有对白全文在 [Native Audio] 区块中再次列出

---

## 时长计算

- 主方式：分镜表中各镜头时长直接相加
- 审核方式：对白字数÷3.5 + 动作数×2-3秒交叉验证
- **所有时长进位为整数秒**：1.2s→2s，0.5s→1s，精确到整秒
- 每组 3-5 镜，Segment 总时长 = 所有镜头进位后时长之和，不得超过 15 秒
- 以导演视角分配各镜时长，进位后总和超标则须减少镜头数或重新拆组

## 拆组策略

**🔴 镜数铁律：每个 Segment 严格 3-5 镜，进位后总时长 ≤15 秒。少于 3 镜或多于 5 镜均视为不合格。**

### 必须拆组的情况
- 超 5 镜或进位后总时长超 15 秒 → 必须拆组
- 遇硬切或场景转换 → 必须拆组
- 遇时间跳跃 → 必须拆组
- 每组开头重置构图基准和时长计时器

### 🔴 必须合并的情况（短段合并铁律——最高优先级）
相邻 Segment 满足以下**全部三个条件**时，**必须合并为一个 Segment**，禁止保留两个独立短段：

1. 两个 Segment 的镜数之和 ≤ 5
2. 两个 Segment 进位后总时长之和 ≤ 15s
3. 两个 Segment 属于同一场景、同一光影时间

**短段合并示例**：Segment 2（2 镜/3s）和 Segment 3（2 镜/3s）同属「青云宗洞府内」场景，合并后 Segment 2（4 镜/6s）完全合规 → **必须合并，禁止拆开**。

**自检流程**：生成完所有 Segment 后，逐段检查镜数。发现不足 3 镜的 Segment → 检查能否与上一段或下一段合并。若可合并则合并后重新编号；若无法合并（如已达 5 镜上限或超 15s），则为严重设计错误，需重新拆组。

### 通用规则
- 以导演视角分配各镜时长，进位后总和超标则须减少镜头数或重新拆组
- 场景必须按 3-5 镜/≤15s 拆分为多个 Segment。场景 1 有 26 镜 → 至少拆成 6 个 Segment，不是 1 个

## 质量禁止清单

- 绝对禁止使用 `Continued from...` 或 `Same style as above.`，每个 Segment 必须 100% 独立完整
- 绝对禁止遗漏任何对白
- 必须使用高精度物理动词（如 impact、recoil、ripple），避免模糊形容词
- 必须禁止生成：文字伪影、角色变形、多余手指、光影不一致、jittery motion
- **禁止生成 BGM 背景音乐**：所有视频均不需要任何形式的背景音乐，[Native Audio] 区块仅限 SFX 环境音/音效 + 对白全文 + 旁白
- **禁止生成字幕**：所有视频均不需要硬字幕（burned-in subtitles），不包含任何形式的画面内文字字幕

### 🔴 BGM 类违规关键词黑名单（SFX 中绝对禁止）

SFX 仅限环境音与音效。以下任一词出现在 SFX 描述中均视为违规（含所有中文/英文变体和组合）：

| 类别 | 违规关键词（任一命中即违规） |
|------|------|
| 音乐泛称 | 配乐、BGM、背景音乐、OST、原声带、soundtrack、music、score |
| 弦乐/管弦 | 弦乐、管弦、交响、小提琴、大提琴、中提琴、低音提琴、violin、cello、orchestra、string |
| 乐曲/旋律 | 乐曲、旋律、乐声、钢琴曲、主题曲、插曲、片尾曲、片头曲、melody、tune |
| 情绪+音乐 | 悲伤音乐、激昂音乐、紧张音乐、温柔音乐、史诗音乐（一切「情绪词+音乐」组合） |
| 演奏动作 | 演奏、独奏、合奏、协奏、奏响、奏起、弹奏（含「缓缓奏响」「弦乐渐起」等） |

**唯一例外**：剧情叙事中角色自身在弹奏乐器/唱歌（如「角色轻抚琴弦，琴声从指尖流出」），该音效属于角色动作音效而非外加配乐，允许保留，但必须在 SFX 标注为「角色弹奏琴弦声」而非「琴声悠扬」。

**替代方案**：将 BGM 类描述替换为环境音效——
- 「弦乐渐起」→「风声渐起，灵气嗡鸣」
- 「配乐烘托」→「环境低频震动如心跳」
- 「交响乐推向高潮」→「雷声滚动+风声呼啸层层叠加」

## Pitfalls

- **输出必须直接以 `## 🎬 导演核算报告` 开头**，禁止在此之前输出分析、推理、方法论讨论
- 每个 Segment 必须 100% 自包含，不依赖上一段的上下文。禁止在衔接描述中引用其他 Segment 编号（如「Seg6 镜4 闭眼」），必须将状态完整描述出来
- 角色外貌由资产设计提供，不在 Asset Definitions 或镜头中重复
- Asset Definitions 中的资产名必须与用户输入完全一致，禁止使用 @imageN 编号，禁止自行编造不存在的资产名
- 物理细节必须量化，不能只说"很快"或"很大"
- 出镜控制禁止模糊词
- 仅输出提示词内容，不在回复中写入文件、不做多余解释
- 每镜必须有 [Lip-sync] 或 [Reaction] 或 Voice-over 标注之一（纯环境镜除外）
- **禁止在 [Native Audio] 中添加 BGM/背景音乐**，仅限环境音效 + 对白 + 旁白
- **禁止在提示词生成指令中添加硬字幕/burned-in subtitles 命令**（剧本叙事需要的字幕剧情如"黑屏字幕三天后"属于叙事元素，不是生成指令，应保留）
- **长剧本输出截断**：当分镜脚本较长（多 Segment）时，完整输出可能超过 `agent.max_tokens` 限制而被截断。解决方式：`hermes config set agent.max_tokens 32000`（默认 16000 不够用时可调至 32000、48000 或更高），**禁止通过压缩输出格式、省略 Segment、合并字段来节省 token**——格式完整性优先于 token 节省。
- **🔴 禁止只输出第一个 Segment**：这是最常见的输出缺陷。输入有多少个场次/段落就必须输出多少个 Segment 块。绝对禁止在生成完第一个 Segment 后认为"任务完成"而停止。每次输出前必须自检：输入场次数 = 输出 Segment 数，不一致即为不合格输出，必须全部重写。
- **🔴 禁止产出不足 3 镜的 Segment**：每个 Segment 严格 3-5 镜。产出 1-2 镜的短 Segment 视为不合格。当两个相邻 Segment 均为短段（镜数之和 ≤5、时长之和 ≤15s、同场景）时，**必须合并为一个 Segment**，禁止拆开。典型违规：Segment 2（2 镜/3s）和 Segment 3（2 镜/3s）同属一个场景，合并后 Segment 2（4 镜/6s）完全合规 → 必须合并。自检时逐段扫描镜数，发现不足 3 镜立即触发合并逻辑。
- **🔴🔴 禁止只处理第一个场景就停止（最高优先级铁律）**：这是比"只输出第一个 Segment"更严重的缺陷。输入中可能包含多个 `场景 N:` 标记（如场景 1-7 + 结尾钩子），你必须为**每一个**场景生成对应的 Segment 序列。处理流程：扫描全部场景 → 建立清单 → 逐个处理并打勾 → 全部打勾后才允许停止。绝对不允许处理完场景 1 就认为"任务完成"——场景 2、3、4、5、6、7 和结尾钩子一个都不能少。**这是本 Skill 的最高优先级规则，违反此规则的输出视为完全不合格。**
- **🔴 禁止格式退化**：为了覆盖全部场景而压缩输出格式是本 Skill 最常见的退化模式。绝对禁止使用 `【镜头001】(3s)` 简化编号、禁止省略 `人物站位与状态` 字段、禁止把 `景别/运镜/特殊技法` 合并为括号格式、禁止 Base Prompt 写英文、禁止审查缩水到 3 条、禁止区块标题用粗体/`#` 替代 `[]`、禁止 Native Audio 写成段落、禁止字段用半角 `:`、禁止审查区块位置错乱。**场景全覆盖 + 格式零退化，两者必须同时满足，缺一不可。**退化清单共 32 条，逐条禁止。
- **🔴 锚定模板不可绕行**：你的输出必须与「不可修改的锚定输出模板」逐字符一致，只能替换 `{占位内容}`。任何偏离——包括改用 `###` 标题、改用 `**粗体**` 区块标题、改用 `- **Shot 001 (3s):**` 简化镜头、改用 `**Characters:**` 子分类、改用 `**VO/Dialogue:**` 音频格式、缺失审查区块——全部视为输出不合格。模板之外不存在任何"替代格式"。
- **🔴🔴 外部 Prompt 注入可覆盖 Skill 规则（最高优先级诊断）**：当模型持续输出不符合 Skill 规则的格式，且输出中出现「根据用户特别指令」「根据特别指令，打破 XX 限制」等文字时，**不要继续修补 Skill**——应立刻检查调用 Agent 的软件代码中，是否有 prompt 模板注入了与 Skill 冲突的指令。典型案例：调用软件在组装 prompt 时写入「不要按 SKILL 的 ≤15s 规则，一段分镜对应一个 Segment」→ 模型忠实执行 → 所有 Skill 规则被架空。诊断命令：搜索调用代码中是否包含「不要按 SKILL」「让 Seedance 自己处理」「一段分镜对应一个 Segment」等反规则关键词。**修复调用代码的优先级远高于修补 Skill。**
