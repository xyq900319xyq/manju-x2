# Seedance-Prompt-Generator 版本变更日志

## v3.2.0 (2026-07-10)

### 修复四大硬伤（基于用户提供的 38-Segment 输出质量审计）

#### 硬伤 1：审查格式缩水
- **现象**：审查二/四/五输出仅 `✅ 审查二通过`，缺详情
- **根因**：结论格式规范中"通过"分支只写了短句，模型照抄导致缩水
- **修复**：审查二/四/五的结论格式全部补齐详情文本

#### 硬伤 2：大量 Segment 不足 3 镜（8 个违规段）
- **现象**：Segment 2(2镜)、3(2镜)、14(1镜) 等多个短段
- **根因**：拆组策略只定义了"何时拆"（超 5 镜/15s），没有"何时必须合并"规则
- **修复**：新增「短段合并铁律」——相邻 Segment 镜数和≤5、时长和≤15s、同场景→必须合并

#### 硬伤 3：SFX 混入 BGM 元素（5 处违规）
- **现象**：SFX 中出现"宏大开场配乐渐起""弦乐渐入""交响乐推向高潮"
- **根因**：质量禁止清单只说"无 BGM"，未列具体关键词黑名单
- **修复**：新增 5 类 20+ 关键词黑名单（配乐/BGM/弦乐/交响/乐曲/旋律/OST 等）

#### 硬伤 4：Segment 2 和 3 该合未合
- **现象**：Seg2(2镜/3s) 和 Seg3(2镜/3s) 同场景可合并为 4镜/6s，却被拆开
- **根因**：硬伤 2 的同源问题，建模时缺少"相邻短段合并"逻辑
- **修复**：短段合并铁律直接解决

### 其他改动
- 退化清单从 30 条扩至 32 条（#31 短段不足 3 镜，#32 BGM 关键词违规）
- Pitfalls 新增"禁止产出不足 3 镜的 Segment"
- 锚定模板强制规则 #10 从 "≤5 镜" 改为 "3-5 镜"
- 审查二标题从"时长可行性"改为"时长与镜数可行性"
- Phase 1 拆组判断新增"不足 3 镜→必须合并"引用
- 版本号升至 3.2.0
