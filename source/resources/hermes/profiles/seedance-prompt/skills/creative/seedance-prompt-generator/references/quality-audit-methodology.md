# 输出质量审计方法论

基于 v3.2.0 版本升级中的实际诊断经验，总结一套系统化的输出质量审计流程。

## 审计流程（四步法）

### Step 1：逐项对照锚定模板
将模型输出与 SKILL.md 中的锚定模板逐字段比对：
- 审查区块：七条完备性 + 每条结论格式完整性
- Segment 区块：标题格式、Asset Definitions、Global Style 后缀
- 镜头块：5 字段完整性、编号格式、时长格式
- Native Audio：5 项逐行、SFX 内容合规

### Step 2：全量扫描数值合规
- 逐 Segment 计数器：镜数（应为 3-5）、进位后总时长（应 ≤15s）
- 相邻短段检测：镜数和≤5 且时长和≤15s 且同场景 → 应合并
- 对白时长核算：字数÷3.5 + 动作数×2-3s 交叉验证

### Step 3：关键词黑名单扫雷
- BGM 类：配乐/BGM/弦乐/交响/乐曲/旋律/OST/soundtrack/score 等
- 政治类：现实国名/政治人物/国旗国徽
- 暴力类：鲜血/血腥/尸体/肢解/枪支/爆炸
- 自残类：自杀/自残/割腕

### Step 4：根因溯源（不问"错在哪"，问"为什么会错"）
每个发现的问题都要追溯到规则层面的漏洞：
- ❌ "审查二缩水了" → ✅ "审查二的结论格式规范中，通过分支只写了短句，模型无法自行补全"
- ❌ "Seg2 和 Seg3 没合并" → ✅ "拆组策略只定义了何时拆，没有定义何时必须合"
- ❌ "SFX 有配乐" → ✅ "质量禁止清单只说'无 BGM'，未列关键词黑名单"

## 典型缺陷→根因映射

| 缺陷现象 | 大概率根因 | 修复方向 |
|---------|-----------|---------|
| 审查格式缩水 | 结论格式→通过分支缺详情 | 补齐各审查通过格式 |
| Segment 不足 3 镜 | 缺镜数下限规则 + 缺合并逻辑 | 新增合并铁律 |
| 大量 1-2 镜短段 | 同上 + 情绪分镜过度拆分 | 合并 + 提醒 3 镜底线 |
| SFX 混 BGM | 缺关键词黑名单 | 列举所有 BGM 类禁用词 |
| 相邻短段未合并 | 纯拆组逻辑，无合并逻辑 | 互补：拆+合双重规则 |
| 审查总数 < 7 | 模板示例中审查格式缩水 | 锚定模板先行修复 |
