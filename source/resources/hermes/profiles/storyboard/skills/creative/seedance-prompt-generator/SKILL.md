# Seedance 2.0 视频提示词生成器

将分镜脚本和资产设计转化为 Seedance 2.0 可执行的视频生成指令。输出遵循统一的 Director Angel 工业化编译格式。

参考文档：
- `references/director-angel-format.md` — Director Angel 原始编译格式（历史参考）
- `references/seedance-industrial-architect.md` — 工业化制片架构师全套设定（历史参考）
- `references/conflict-analysis.md` — 两份文档的逐字段冲突对照（合并依据）

## 触发条件

- 用户要求为分镜生成 Seedance 视频提示词
- 用户提供分镜内容 + 资产描述，要求生成视频指令
- 用户说"编译成 Seedance 格式"或"生成视频提示词"

---

## 工作流程

### Phase 1: 时长核算

在处理输入的分镜脚本时，自动执行以下核算：

1. **时长审核**：分镜表中各镜头时长直接相加得到总时长，同时用对白字数÷3.5 + 动作数×2-3秒交叉验证
2. **拆组判断**：每组 3-5 镜，总时长 ≤15 秒。以导演视角确定每个镜头的时长分配。超 15 秒或超 5 镜按以下规则拆组：
   - 遇硬切或场景转换 → 必须拆组
   - 遇时间跳跃 → 必须拆组
   - 每组开头重置构图基准
3. **核算报告**：生成前先输出导演核算报告（总时长预估 + 拆段方案）

### Phase 2: 全局初始化

1. 提取并记住所有核心角色、场景、道具特征
2. 读取用户输入中的资产列表（人物、场景、物品），在 Asset Definitions 中使用「资产名：@资产名」格式列出。资产名必须与输入完全一致，不得自行编造。同一光影时间的场景采用统一命名（如 `合欢宗大殿-日`、`柳如烟卧室-夜`）

### Phase 3: 连贯性保障

跨段衔接强制执行：

- **视觉桥梁**：Segment[N].Shot1 锁定 Segment[N-1].Shot5 的机位/角色姿势/光影/环境状态，但描述时必须完整写出状态而非引用编号
- **视觉保戏**：Segment [N] 的 Shot 1 必须完整描述 Segment [N-1] 的 Shot 5 结尾状态，而非引用编号。禁止写「Seg6 镜4 闭眼」，必须写为「延续前段——师尊双眼紧闭，面部肌肉松弛，嘴角残留血迹，靠在洞壁上」。
- **构图基准**：组内镜头2继承镜头1的摄像头位置、FOV参数
- **轴线规则**：对话轴线 180° 不跨轴，动作轴线方向固定
- **站位摘要**：角色相对位置明确（如"距机器2m"）
- **叙事连续性**：对白必须自然跨段流动，长对白配合 Reaction Shots + Voice-over 实现
- **位置状态延续**：每个镜头必须完整描述所有出镜角色的空间位置和当前状态。剧情未推动时直接继承上一镜描述，角色移动后以新位置为基线延续。禁止出现角色无故跳跃到其他位置，禁止状态突变。
- **延续前段标注**：Segment [N] 的 Shot 1 主体动作前标注「延续前段」

### Phase 4: 矩阵生成

每个 Segment 独立输出，100% 自包含。绝对禁止 `Continued from...` 或 `Same style as above.`

**输出规则：你的回复第一个字符必须是 `#`，以 `## 🎬 导演核算报告` 直接开头。禁止在此之前输出任何文字——包括分析、推理、「好的」「让我」等一切非模板内容。你是一个提示词编译器，不是聊天助手。**

## 输出前三重内部审查（仅内部执行，不输出）

正式输出前静默执行三次审查。仅在审查未通过时才在输出中报告问题，通过则直接输出最终提示词。

### 审查一：剧情与台词完整性
逐条核验输入中每个剧情节点是否有对应镜头覆盖，每一句对白是否完整出现并标注 [Lip-sync]，旁白是否标注 Voice-over/O.S。

### 审查二：时长可行性（导演级）
逐镜核算时长是否足够容纳全部内容：对白 3.5 字/秒，动作 2-3 秒/个。铁律：不允许 Segment 结束而对白或动作未完成。

### 审查三：跨镜连贯性（导演级）
逐镜审查人物站位、光影时间、角色状态的连贯性。不得无故漂移、昼夜突变、状态跳变。
- **铁律**：每个镜头必须完整描述所有出镜角色的空间位置和当前状态。剧情未推动时直接继承上一镜描述（如「A仍坐于桌左侧」），角色移动后以新位置为基线延续。禁止跳过位置和状态描述。
- 衔接描述中禁止引用其他 Segment 编号，必须完整描述前段结尾状态

## 输出格式（完整模板）

```
## 🎬 导演核算报告：总时长约 X 秒，拆分为 N 个 Segment。

🎞️ Segment N: 标题 (0-Ys)

[Asset Definitions]
角色A：@角色A
场景B：@场景B
物品C：@物品C
（资产名必须与用户输入中提供的资产名称完全一致，不得自行编造。人物、场景、物品均使用「资产名：@资产名」格式。）

[Global Style]
{从资产设计提取的视觉风格}，4K，cinematic texture，strong character consistency，physics-accurate motion。

[Base Compiled Prompt]
{逐镜头画面描述 + 出镜角色 + 镜头语言 + 动作推进的一段完整文字叙述}

[Director's Shot Matrix]

镜头1 [0s-Xs]「场景名称」：
  出镜角色: {角色名（主/次）或"无角色/环境镜"}
  人物站位与状态: {完整描述所有出镜角色的空间位置与当前状态}。如「A坐于桌左侧木椅上，B立于桌右侧，两人面对面」「师尊靠坐于洞壁，双眼紧闭，嘴角残留血迹」
  主体动作: {角色行为描述}。对白标注 [Lip-sync]，反应镜头标注 [Reaction]，画外音标注 Voice-over/O.S
  镜头语言: 景别 {自由中文}；运镜 {运镜方式}；特殊技法 {技法}
  动作推进: {从状态A→状态B的转变}

镜头2 [Xs-Ys]「场景名称」：
  人物站位与状态: {继承镜头1状态}。若剧情未改变位置和状态则直接延续上一镜描述，若有变动则写出新状态
  ...

[Native Audio]
- SFX：{精准时序的环境音、音效}
- 对白全文：{本段所有台词，标注 Lip-sync}
- 旁白：{Voice-over 内容}
```

### 时长编号铁律

- 所有时长必须是整数秒，使用进位法：1.2s → 2s，3.7s → 4s，2.0s → 2s
- 每个 Segment 的时长从 0s 开始独立计时，不接续上一段的结束时间
- Segment 标题写 `(0-Ys)`，Y = 本段所有镜头进位后时长之和，不得少加
- 每段内镜头编号独立从 1 开始（镜头1、镜头2…），不是全集的连续编号
- 进位后 Segment 总时长不得超过 15 秒。若超过，必须减少镜头数或重新拆组

## 逐字段说明

### Asset Definitions
全量列出本次生成调用的所有资产，格式为 `资产名：@资产名`。资产名必须与用户输入中提供的资产名称完全一致，不得自行编造或使用 @imageN 编号。人物、场景、物品统一使用此格式。

```
林星野：@林星野
废弃工厂黄昏内景：@废弃工厂黄昏内景
档案袋：@档案袋
```

### Global Style
从资产设计中提取视觉风格，可变。常见风格：
- 新海诚风格、电影感、高质量、4K、HDR调色
- Unreal Engine 5 写实渲染风格
- 吉卜力风格、手绘动画质感
- 赛博朋克霓虹美学

如无指定，默认："新海诚风格、电影感、高质量、4K"。

### Base Compiled Prompt
逐镜头画面描述 + 出镜角色 + 镜头语言的完整文字叙述，一段话覆盖本段所有镜头。

### 导演校准要点（内部推理，不输出）
Agent 生成时必须内部执行以下推理，确保跨镜一致性，但最终输出中不出现此模块：

| 字段 | 说明 |
|------|------|
| 组间承接策略 | 当前组与上/下组的关系（独立提交/顺承/呼应） |
| 动作因果 | 前镜动作的后果如何延续到本镜 |
| 同一空间约束 | 同场景镜头必须共享空间参数（尺寸、光线方向、道具位置） |

### 单镜出镜控制（内部推理，不输出）
Agent 生成时必须内部逐镜确认出镜角色，但最终输出中不出现出镜表模块。每镜的出镜角色信息融入「出镜角色」字段中。
- `无角色/环境镜` → 不允许出现任何人物
- `角色名（主）` → 核心焦点
- 出镜控制禁止模糊词（"可能有""包括但不限于"）

### 标准分组连续构图锁定（内部推理，不输出）
Agent 生成时必须内部执行以下连贯性校验，确保跨镜/跨段衔接：

| 字段 | 说明 |
|------|------|
| 构图基准 | 镜头2继承镜头1的摄像头位置、FOV |
| 轴线规则 | 对话轴线 180° 不跨轴；动作轴线方向固定 |
| 站位摘要 | 角色相对位置（左/右/前/后，距离数值） |
- **视觉桥梁**：Segment[N].Shot1 锁定 Segment[N-1].Shot5 的机位/角色姿势/光影/环境状态，但描述时必须完整写出状态而非引用编号

### 单镜字段详解

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| 出镜角色 | string | 角色名或"无角色/环境镜"，含主次 | 林星野（主）-红发金瞳；苏晓（次） |
| 人物站位与状态 | string | 所有出镜角色的空间位置和当前状态 | A坐于桌左侧，B立于桌右侧，两人面对面 |
| 主体动作 | string | 角色行为描述 + 对白标注 | [Lip-sync] 何丹冷笑："违约金从期权扣。" |
| 镜头语言 | string | 景别 + 运镜 + 特殊技法 | 景别 极致特写；运镜 Handheld breathing + 0.1s reflexive jitter |
| 动作推进 | string | 状态A→状态B转变 | 从凝视枪口→缓缓闭眼→嘴角露出释然微笑 |

## 镜头语言术语表

### 景别（Shot Size）
使用自由中文描述，常用词包括：
极远景、全景、中全景、中景、近景、特写、极致特写、瞳孔微距、广角推近、固定远景、极致远景拉远、低角度细节、侧方仰拍

### 运镜（Camera Movement）
- Handheld breathing + 0.1s reflexive jitter — 手持呼吸感晃动
- Snap Zoom — 快速变焦
- Orbit — 环绕运镜
- Reflexive Jitter — 应激性震颤
- Dolly — 推拉
- Pan — 横摇
- Tilt — 竖摇
- Truck — 跟拍平移
- Focus Pull — 焦点切换
- Pedestal/Crane — 升降

### 特殊技法
- 匹配转场（Match Cut）
- J-Cut/L-Cut（声音/画面先入先出）
- 浅景深（Shallow DoF）
- 慢镜头（Slow Motion）
- 快镜头（Speed Ramp）
- Dutch Angle（倾斜构图）
- POV（主观视角）
- 视觉桥梁（Visual Bridge）
- 延续前段（Continuity Anchor）

## 角色出镜控制

- 角色通过 `角色名：@角色名` 格式绑定，外貌和特征由资产设计提供，不在此处重复
- 多角色镜头标注主从关系：「角色A（主）；角色B（次）」
- 出镜控制必须明确，禁止模糊词
- 所有 @ 引用的资产名必须与用户输入中的名称完全一致，不得自行编造

---

## 对白处理

- 对白保留原文，打 `[Lip-sync]` 标签
- 同时附加可视化表演描述（表情、动作、肢体语言）
- 旁白标注 `Voice-over` 或 `O.S`（画外音）
- 长对白配合 Reaction Shots + Voice-over 跨段流动，不得遗漏台词
- 所有对白全文在 [Native Audio] 区块中再次列出

---

## 时长计算

- 主方式：分镜表中各镜头时长直接相加
- 审核方式：对白字数÷3.5 + 动作数×2-3秒交叉验证
- **所有时长进位为整数秒**：1.2s→2s，0.5s→1s，精确到整秒
- 每组 3-5 镜，Segment 总时长 = 所有镜头进位后时长之和，不得超过 15 秒
- 以导演视角分配各镜时长，进位后总和超标则须减少镜头数或重新拆组

- 超 5 镜或进位后总时长超 15 秒 → 必须拆组
- 每组独立完整，开头重置构图基准和时长计时器时长之和，不得超过 15 秒

## 质量禁止清单

- 绝对禁止使用 `Continued from...` 或 `Same style as above.`，每个 Segment 必须 100% 独立完整
- 绝对禁止遗漏任何对白
- 必须使用高精度物理动词（如 impact、recoil、ripple），避免模糊形容词
- 必须禁止生成：文字伪影、角色变形、多余手指、光影不一致、jittery motion

## 输出语言

全中文输出。运镜术语保留英文原样（如 Handheld breathing、Snap Zoom 等），景别使用自由中文。

## Pitfalls

- **输出必须直接以 `## 🎬 导演核算报告` 开头**，禁止在此之前输出分析、推理、方法论讨论
- 每个 Segment 必须 100% 自包含，不依赖上一段的上下文。禁止在衔接描述中引用其他 Segment 编号（如「Seg6 镜4 闭眼」），必须将状态完整描述出来
- 角色外貌由资产设计提供，不在 Asset Definitions 或镜头中重复
- Asset Definitions 中的资产名必须与用户输入完全一致，禁止使用 @imageN 编号，禁止自行编造不存在的资产名
- 物理细节必须量化，不能只说"很快"或"很大"
- 出镜控制禁止模糊词
- 仅输出提示词内容，不在回复中写入文件、不做多余解释
- 每镜必须有 [Lip-sync] 或 [Reaction] 或 Voice-over 标注之一（纯环境镜除外）
