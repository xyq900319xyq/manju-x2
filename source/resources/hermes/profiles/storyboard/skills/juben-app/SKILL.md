## Zip Packaging

Must use embedded Python for zip creation (avoids MSYS path issues):
```python
import zipfile, os
src = r'D:\剧本分镜助手_打包'
with zipfile.ZipFile(dst, 'w', zipfile.ZIP_DEFLATED) as zf:
    for root, dirs, files in os.walk(src):
        dirs[:] = [d for d in dirs if d not in ('__pycache__', 'Scripts')]
        for f in files:
            fp = os.path.join(root, f)
            arc = os.path.relpath(fp, src)
            # DON'T skip .py in hermes/ or runtime/ — Hermes needs its source
            if f.endswith('.py') and 'hermes' not in arc and 'runtime' not in arc and f != 'launcher.py': continue
            if f.endswith('.spec'): continue
            zf.write(fp, arc)
```

**CRITICAL**: The zip MUST be flat (no intermediate folder). Verify zip contents after creation:

- `hermes/hermes.exe` + `hermes/hermes_cli/`

## Launcher.py Code Generation

When writing launcher.py via execute_code, the tool censors and escapes content:
1. **Tokens** get replaced with `...` — read from `_token.txt` and replace after writing
2. **f-string braces** (`{{` and `}}`) get written literally — use patch tool to fix to `{` and `}`
3. **`repr(secret)`** templates get written literally — replace with actual bytes after
4. **NEVER rewrite the entire file** via execute_code once stable — use patch tool for targeted fixes

## Activation System

### Machine ID
Use Windows MachineGuid (registry `HKLM\SOFTWARE\Microsoft\Cryptography\MachineGuid`) — unique per Windows installation, survives hardware changes, cannot be changed by user.

### GitHub Token Auth
Use **Basic auth** (not Bearer):
```python
auth = base64.b64encode(f"x:{token}".encode()).decode()
req.add_header("Authorization", f"Basic {auth}")
```
Token needs "repo" scope (classic token, check all repo sub-items).

### Whitelist flow
1. Codes generated by activation manager → auto-pushed to public repo `codes.txt`
2. User enters code → launcher verifies HMAC + checks public `codes.txt`
3. On success → reports machine code to private repo `activations.txt` via API
4. Report MUST succeed before activation completes

### Heartbeat timeout
Default was 15s — too short when user fills forms. Changed to 60s.
```python
def _heartbeat_watcher():
```

### Server-side profile discovery & API injection (CRITICAL)
The server must NOT rely on launcher env vars to find decrypted profiles. It must auto-discover:
```python
def _find_profiles_base():
    base = os.environ.get("HERMES_PROFILES", "")
    if base and os.path.exists(os.path.join(base, "storyboard", "config.yaml")):
        return base
    import tempfile
    for d in os.listdir(tempfile.gettempdir()):
        if d.startswith("MSI") and os.path.exists(os.path.join(tempfile.gettempdir(), d, "storyboard", "config.yaml")):
            return os.path.join(tempfile.gettempdir(), d)
    return os.path.expanduser("~/.hermes/profiles")
```

And inject API config BEFORE every Hermes call:
```python
def _inject_api_to_profile(profile_dir):
    api_file = os.path.join(os.environ.get("USER_DATA") or 
        os.path.join(os.environ.get("APPDATA", os.path.dirname(__file__)), "Juben"), "hermes_api.json")
    if os.path.exists(api_file):
        cfg = json.load(open(api_file))
        yp = os.path.join(profile_dir, "config.yaml")
        if os.path.exists(yp):
            d = yaml.safe_load(open(yp))
            m = d.setdefault("model", {})
            m.update({k: cfg[k] for k in ("api_key","base_url","provider") if k in cfg})
            m["default"] = cfg.get("model", m.get("default", "deepseek-v4-pro"))
            yaml.dump(d, open(yp, 'w'), allow_unicode=True)
```

```python
def _safe_migrate():
    conn = get_db()
```
