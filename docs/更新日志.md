## v1.1.5.2 (2026-07-10) — 批量资产生图 SQLite 跨线程修复

### 🟠 高严重修复
- **批量资产生图失败**:`BatchAssetImageTask.run()` 在 task queue worker thread 调
  `self._db.list_assets/update_asset_image/update_asset_status`,但 `self.db` 的
  `sqlite3.Connection` 是 main thread 创建的(`main_window.py:118` → `open_db(ROOT)`),
  SQLite3 默认 `check_same_thread=True` → 跨线程报 "SQLite objects created in a
  thread can only be used in that same thread. The object was created in thread
  id 14720 and this is thread id 14960."
  修法:`core/migration.py open_db` 加 `check_same_thread=False`。SQLite 内部有
  mutex 保护并发安全,task queue 串行跑无 race。
  其他 5 类 task(`StoryboardTask` / `VideoPromptTask` / `AssetExtractTask` /
  `AssetImageTask` / `VideoTask`)通过 hermes 处理不直接读 db 不受影响。

## v1.1.5.1 (2026-07-10) — 紧急修复:hermes 三个智能体打包丢失

### 🔴 致命修复
- **三个智能体(资产提取 / 提示词 / 分镜)安装后加载失败,hermes 输出 "skill 未在系统中安装"**:
  `build_x2.py` step 3 只把 `hermes.exe` 拷到 `dist/.../hermes/`,**漏拷** `source/resources/hermes/profiles/` 到 `dist/.../resources/hermes/profiles/`。
  后果:EXE 安装后 `<install_root>/resources/hermes/profiles/` 不存在 → `Config.hermes_home` 探测 fallback 到 `<hermes.exe>/../` 也没有 profiles/ → `ensure_local_hermes_profile` 写最小 config.yaml(没 SOUL.md / SKILL.md)→ hermes 跑时找不到 skill。
  修法:build_x2.py step 3 之后加一步 copytree,完整拷 asset-designer / seedance-prompt / storyboard 三个 profile 到 `dist/.../resources/hermes/profiles/`(含 skills/ 几百个 SKILL.md)。

## v1.1.5 (2026-07-10) — 16 个 BUG 全面修复

### 🔴 致命修复
- **导入剧本按钮崩溃**:`_on_import_script_file` 调了不存在的 `_show_storyboard_tab`,导致点"📥 导入剧本" → AttributeError → Qt 弹"内部错误"。整个导入剧本功能是坏的 → 改用 `_show_episode_detail` + invalidate cache

### 🟠 高严重修复(6 个)
- **新建项目不自动选中**:点"新建项目"后 toolbar "新建剧集" 按钮还是灰的,user 不知道为啥按不动 → `add_project` 返回 item + `setCurrentItem` 触发 selection
- **新建剧集不自动跳到新剧集**:user 期望看到新剧集的分镜 tab,实际还停在原剧集/项目概览 → `add_episode` 返回 child + `setCurrentItem`
- **删除项目后右侧 tab 仍显示已删剧集**:`_current_project` / `_current_episode` 不清空 → 切回空态 + invalidate cache
- **删除剧集后项目行括号 `(N)` 数字不更新**:`remove_episode` 不会更新 → `set_episodes` 重画括号数字
- **点"取消分镜"后 hermes 子进程跑满 2 小时**:`StoryboardTask` 漏 override `cancel()`,对比 `AssetExtractTask` / `VideoPromptTask` 都有 → 补上 `mt.cancel()` 透传
- **创维等环境拉不到 update.json**:`updater.py` 漏 `ssl._create_unverified_context()`,对比 `image_api.py` / `generators.py` 都有 → 三处全加
- **资产 image_prompt 切走再回来被回滚**:`_on_prompt_changed` 写库后不更新 `self._asset` 内存 → 写库后 in-place 更新 dataclass + 通知 listw item

### 🟡 中严重修复(10 个)
- **重命名项目后概览 tab 仍显示旧名**:`_project_overview_cache` 命中 → 改名后清 cache + 主动刷
- **编辑剧集后各 tab 显示旧 title/script**:`_invalidate_all_ui_caches()` 后主动重画
- **资产生图/批量生图/生视频失败时不弹"换模型"对话框**:`_maybe_prompt_model_switch` 只对 3 类 LLM task 弹 → 扩到 6 类
- **空 image_prompt 直接发给生图 API**:有些平台返回透明图/随机图 → 源头抛 RuntimeError,提示补全
- **拉取模型列表网络异常被吞**:用户看不到任何错误 → 加兜底 `except Exception` 弹错误框
- **测试 dreamina.exe 异常裸抛**:`_on_test_dreamina` 裸调 subprocess → 加 try/except
- **dreamina OAuth 静默吞 `_dreamina_logged_in` 异常** + 启动异常裸抛 → 加 `log.warning` + try/except
- **删项目/剧集时 `itemSelectionChanged` 误触**:`takeChildren/removeChild/takeTopLevelItem` 触发 selection → blockSignals 包起来
- **`task_queue` emit 信号异常被吞**:UI 状态卡住不知任务在跑 → 4 处加 `log.warning/exception`
- **数据迁移 / b64 decode / comdlg32 异常被静默吞** → 加 `log.debug` 留痕迹
