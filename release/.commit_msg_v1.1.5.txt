v1.1.5 — 16 个 BUG 全面修复(1 致命+6 高+10 中)

## 修复
- 致命:导入剧本按钮崩溃(AttributeError: _show_storyboard_tab 不存在)
- 高:新建项目不自动选中 / 新建剧集不自动跳 / 删项目/剧集状态不清空
- 高:StoryboardTask 漏 cancel override(取消后 hermes 跑满 2 小时)
- 高:updater.py 漏 ssl._create_unverified_context()(创维环境)
- 高:资产 image_prompt 切走再回来被回滚
- 中:重命名/编辑剧集 cache 命中显示旧值
- 中:资产生图/批量生图/生视频失败不弹换模型对话框
- 中:空 image_prompt 直接发 API / 拉取模型异常被吞
- 中:dreamina 测试/OAuth 异常被吞
- 中:删项目/剧集触发 itemSelectionChanged 误触
- 中:task_queue 4 处 emit 异常被静默吞

## 根因
1. `_on_import_script_file` 引用了不存在的 `_show_storyboard_tab`
   (整个文件 0 处定义),改成 `_show_episode_detail`
2. `_on_new_project` / `_on_new_episode` 用 `addTopLevelItem` / `addChild`,
   Qt 不会触发 `itemSelectionChanged`,user 新建完 tab 不动
3. `_on_delete_project` / `_on_delete_episode` 不清空 `_current_*`,
   删完右侧 tab 还显示已删内容
4. `StoryboardTask` 漏 override `cancel()`(对比 `AssetExtractTask` /
   `VideoPromptTask` 都有 `mt.cancel()` 透传)
5. `updater.py` 3 处 `urlopen` 没加 `ssl._create_unverified_context()`,
   自签证书环境拉不到 update.json(创维 / 部分公司反代)
6. `asset_panel._on_prompt_changed` 写库后不更新 `self._asset` 内存,
   切走再回来被 listw 旧对象覆盖
