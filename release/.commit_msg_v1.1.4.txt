v1.1.4 — 资产提取/资产生图 UI 不刷新统一修复

## 修复
- 资产提取成功了但 UI 不显示(同 v1.1.2 分镜/提示词 UI 不刷新同类问题)
- 新版 hermes 资产输出格式 `# 一、人物资产` 解析失败
- 空 desc 资产被 `if desc:` 过滤掉

## 根因
1. `_persist_task_result` 入口没统一 invalidate UI cache。v1.1.2 只对
   StoryboardTask / VideoPromptTask 加了 `_episode_detail_cache = None`,
   资产提取 / 资产生图 / 批量生图完成后 `_asset_tab_cache` 仍是 id-only
   命中 → UI 永远显示老"暂无"
2. `parse_asset_markdown` 段标题正则只认 `## 人物资产` 老格式,新版 hermes
   输出用 `# 一、人物资产` / `# 二、场景资产` / `# 三、物品资产` 带中文
   序号 → 整段被跳过,`result_assets = []` → db 没数据
3. `### 物品4：xxx` 后直接换 `### 物品5`,desc 空被 `if desc:` 过滤掉

## 改动
- `ui/main_window.py` 新增 `_invalidate_all_ui_caches()` helper(失效
  `_episode_detail_cache` / `_prompt_tab_cache` / `_video_tab_cache` /
  `_asset_tab_cache` / `_project_overview_cache`),`_persist_task_result`
  入口处统一调用。任何 task 完成 → 强制失效 → 下次切 tab 重建/刷新
- `core/asset_parser.py` 段标题正则兼容 `# 一、人物资产` /
  `## 一、人物资产` / `# 1. 人物资产` 等带序号格式,旧 `## 人物资产`
  也保留
- `core/asset_parser.py` 空 desc 资产也保留(只 name 没 desc 也 upsert)
- `installer/漫剧助手X-2.iss` `OutputBaseFilename` 改纯 ASCII
  `X-2_v{#MyAppVersion}_Setup`(v1.1.3.1 起 GitHub 截断中文)
- `build_x2.py` glob pattern 兼容老 "漫剧助手X-2_v*" + 新 "X-2_v*";
  step 7 按 semver 排序(字符串排序 "X" < "漫" 错位取错 latest)
- 版本号 1.1.3 → 1.1.4 (main.py 2 处 + main_window.py 1 处 + .iss 1 处)
