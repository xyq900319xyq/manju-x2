v1.1.5.2 — 批量资产生图 SQLite 跨线程修复

`BatchAssetImageTask.run()` 在 task queue worker thread 调
`self._db.list_assets/update_asset_image/update_asset_status`,但 `self.db`
的 `sqlite3.Connection` 是 main thread 创建的(`main_window.py:118` →
`open_db(ROOT)`),SQLite3 默认 `check_same_thread=True` → 跨线程报
"SQLite objects created in a thread can only be used in that same thread."

修法:`core/migration.py open_db` 加 `check_same_thread=False`。SQLite 内部
有 mutex 保护并发安全,task queue 串行跑无 race。

其他 5 类 task(StoryboardTask / VideoPromptTask / AssetExtractTask /
AssetImageTask / VideoTask)通过 hermes 处理不直接读 db 不受影响。
