## 已知问题(按优先级)

### P0 (阻塞性,装/用就出问题)

#### Bug-1: ChineseSimplified.isl 缺失,中文用户装好看到英文 UI
**症状**: 中文用户装好 Setup.exe,首次弹窗全英文(license 页 / 安装页)
**原因**: `D:\InnoSetup6\Languages\ChineseSimplified.isl` 缺(用户装机时漏勾 Chinese)
**修法**: v1.0.1 Setup.exe 改用内置 `compiler:ChineseSimplified.isl`(要求 ISCC 安装时勾 Chinese),或者打包 `installer\Languages\ChineseSimplified.isl` 随 .iss 一起
**优先级**: 🔴 P0(中文用户全受影响)
**负责人**: dev 端构建时
**验证**: 干净 Win11 VM 装,看 license / 安装页是否中文

#### Bug-2: 5 个 VM 端测试项 (Phase 6 残留)
来自 C5 / C12 / C13 / C14,需要在干净 VM 跑:
- C5: 无 UAC 装(默认路径 C:\漫剧助手X-2\)
- C12: Setup.exe 卸载(控制面板 → 卸载)
- C13: Setup.exe 覆盖升级
- C14: EXE 双击启动 + 弹 wizard
- C15: 真 GitHub release 自动检查(发版后跑)

**修法**: 灰度用户报告 → 改 → 验证
**负责人**: 发版人(去 VM 跑)
**优先级**: 🟡 P1(发版前最后一道关)

### P1 (一般 bug,影响使用但能 workaround)

#### Bug-3: hermes agent 行为问题 - "1:1 拆段" 约束
**症状**: 分镜生成时,hermes agent 主动选择只输出第 1 场景(75秒)而不是全部 170秒
**原因**: `source\src\core\prompts.py:458` 第 4 条 "1:1 拆段" 约束 + hermes 端 SKILL 没强制要求"剧情完整性"
**修法**: 加 `hermes --resume` 续传机制(进程层补全),或改 prompts.py 移除"1:1 拆段"
**优先级**: 🟡 P1
**状态**: user 之前没明确撤回 prompts.py:458
**待 user 决策**

#### Bug-4: DeepSeek thinking 模式消耗 token 导致分镜截断
**症状**: 第 2 集只生成 2 个 Segments(应 30 个)
**原因**: DeepSeek V3.1 thinking mode 消耗 token + 16K max_tokens 限制
**修法**: hermes 端处理(关 thinking / 换非 thinking model / 加 max_tokens),manju 不改
**负责人**: hermes 端
**优先级**: 🟡 P1(影响重度用户)

#### Bug-5: 视频提示词生成只输出前 12/30 Segments
**症状**: v0.7.8.76 已加续传机制,但 DeepSeek thinking 模式触发后静默截断,检测不到截断标记
**修法**: 已部分修(`_call_hermes_with_retry`),v1.0.1 验证修好
**优先级**: 🟡 P1

#### Bug-6: 安装时报"另一个安装正在进行"
**症状**: 用户电脑里有其他 Setup.exe 在跑,装不上
**修法**: 加 retry + 提示用户关掉之前的安装进程
**优先级**: 🟢 P2(罕见,workaround 简单)
